# アートディレクション

## 共通基準

承認済みの夏の住宅街コンセプトを、カメラ、画材、光、色、密度の基準とする。

- M1.4正式基盤は高解像度の横視点2D
- M1.3の斜め上からの固定2.5Dはフォールバックとして保存
- 日本の2000年代前後の夏休み
- 瓦、壁、路面、葉のペインタリーな質感
- 暖色の直射光と青緑の影
- 道路・歩道・私有地が一目で判別できる構図

## M1.3

4区間は反転した同一画面ではなく、主人公の家の前、生活道路、路地と曲がり角、自販機交差点として独立構図を持つ。歩行ルートを先に設計し、その後に家、塀、植栽、小物、前景を配置する。

主人公は4方向で同一人物に見える輪郭・色・服装を維持し、8フレームの歩行で足・腕・体重移動を表現する。背景と主人公の品質差をVisual Evidenceで確認する。

## M1.4 — 2D横スクロール（完了・Production確認済み）

M1.3までの斜め見下ろし2.5Dは設計履歴とProductionフォールバックとして残し、M1.4のメイン表現は高解像度2D横視点へ切り替える。

### 共通トーン

- 日本の夏休み、暖かくノスタルジックな住宅街
- ペインタリーな住宅、塀、アスファルト、植栽、小物
- 遠景、中景、地面、前景で奥行きを作り、主人公の地面ラインを崩さない
- 朝は柔らかい生活感、昼は強い夏空、夕方は暖色と長い影、夜は窓・街灯・自販機の光
- 主人公、背景、UIの輪郭、解像感、光方向、色温度を同じゲームとして統一する

### 3エリアの差別化

- `home-street`: 主人公の家を視覚的な起点にし、隣家、塀、植え込み、電柱を置いた静かな道路
- `life-road`: 住宅の連続、門、自転車、植木鉢、道路標識、電線と、上へ抜ける細い分岐
- `upper-vending-lane`: 古い塀、物置、濃い木陰、オリジナル自販機を持つ少し高い住宅路地

3エリアは別々のmasterと構図を持つ。同じ背景の反転、複製、引き伸ばしで済ませず、住宅配置、道路形状、小物、植栽、空の抜け、前景シルエットを変える。

### 主人公

M1.4専用の横視点主人公を用いる。左右各4フレームの待機と10フレームの歩行、合計28フレームを基準にする。左右の足と腕を逆位相にし、身体の上下動を抑え、接地フレーム2・7が地面と一致して見えるようにする。足元の影は別レイヤーとし、停止中や画面端で足踏みさせない。

### UI

上・下矢印は、背景の一部と誤認しない明瞭な形、方向ラベル、十分なコントラストを持つ。タッチ端末では全体を押せる大きさにし、街の景観と主人公を過度に隠さない。既存ゲームのUI意匠は複製しない。

### 品質判定

生成物は説明用画像ではなく実ゲームへ組み込んだ。PR #32とmain `147f770a4b73077c4e5dc0523839b3fefb789db4`のProduction Artifactでは、3エリア、左右歩行、上下矢印、4時間帯、スマートフォン横画面のシナリオを確認した。この配信・Artifact履歴は維持する。

ただし既存の接地invariantは主人公Yと実装上の`groundY`の一致であり、背景道路面を独立した正解として測ったものではない。2026-07-23の[ユーザー実機所見](evidence/M1_5_REAL_DEVICE_FINDINGS.md)で、主人公の未完成感、空中歩行、背景と分岐triggerのずれ、背景に存在しない下出口、遷移パネルとの重なりが確認された。このためM1全体の品質判定を再オープンし、次の基準をM1.5の必須gateとする。

## M1.5 必須アート修正

### 主人公

- 背景と同等の完成度を持つ最終ラスタ素材にする
- 少年の設定、服装、日本の夏休みの世界観を維持し、ペインタリー背景へ馴染ませる
- 左右idle／walk、停止、向き変更の全28フレームで足元pivot、顔、体格、輪郭、服装、光方向を一貫させる
- atlasとfoot pivotを最終素材から再計測し、影は身体と別レイヤーにする
- 透過縁、身体や服の切断、halo、足元の欠けを原寸とゲーム内表示で目視確認する
- 3エリア、4時間帯、左・中央・右で浮遊、埋没、足滑りを発生させない
- 出所、生成方法、生成・修正日、使用ツール、人手修正、権利・ライセンス、最終ファイルのSHA-256を記録する

### 背景、接地、分岐

- 3エリアを背景から独立注釈したfixtureを唯一の出典とし、runtime値を期待値へ自己参照しない
- エリアごとにwalkable領域、背景から採ったground line、spawn、背景上の分岐入口、triggerを同一座標系の表へ記録する
- 開始、中央、終端、spawn、分岐で、表示上の足裏と背景道路面の差を原則2 CSS px以内にする
- spawn設定点のYと独立groundとの差を6 CSS px以内にし、spawn後の表示上の足裏は2 CSS px原則でも確認する
- 背景変更時はground、spawn、入口を再注釈・再計測する
- `life-road`は背景上の上り道へ到達した時点で上案内を表示する
- `upper-vending-lane`は下矢印に対応する道、坂、階段、開口部のいずれかを背景へ描く
- 背景入口中心とtrigger中心の差を5 CSS px以内にし、prompt表示中だけ上下遷移入力を有効にする
- 見えている道と内部triggerが矛盾する構図を合格にしない

### UIとの構図

- 主人公と遷移パネルの交差を0にし、最短間隔を12 CSS px以上にする
- 遷移パネルとHUDを交差させない
- `1280×720`、`844×390`、`932×430`でsafe-areaを含めて配置する
- タッチ領域を44×44 CSS px以上にし、主人公、足元、進行方向を隠さない
- 上下 × trigger開始・中央・終端 × 左右向きの12状態でplayer／panel／HUD実矩形を測定する

詳細な測定方法と完了gateは[M1.5 必須実機品質修正仕様](specs/M1_5_REAL_DEVICE_POLISH.md)を正とする。同一SHAのPreviewでユーザー実iPhone承認を得るまでmainへ進めず、承認済みSHAのProduction確認までM2を開始しない。
