# 技術アーキテクチャ

## 採用技術

- React 19：会話、ショップ、カレンダー、日記、設定、エンディングなどのUI
- TypeScript：ゲームとUI双方の型安全性
- Vite 6：開発・本番ビルド
- Phaser 4.2.1：マップ、カメラ、キャラクター、衝突、描画、演出
- Vercel：`main`ブランチの本番配信
- GitHub Actions：マージ前の品質チェック
- localStorage / IndexedDB：将来のローカルセーブ

## UIとゲームエンジンの責務

### Phaser

- 2.5Dフィールド描画
- マップストリーミング
- 主人公・NPCの移動
- 衝突判定と描画順
- 自販機・埋蔵金・バス停などのインタラクション
- 時間帯による空、光、影、街灯、窓の変化

### React

- タイトル
- 会話テロップと上半身画像
- 所持金、日付、時刻、ゲーム本数HUD
- ショップ・持ち物・カレンダー
- 日記とエンディング
- 横画面案内、設定、ビルド情報

## M0時点の通信

`CustomEvent`を使い、ReactからPhaserへプレビュー時刻を通知する。M1以降は型付きEventBusへ拡張する。

## 時間帯基盤

ゲーム時刻を分単位の整数で保持し、以下のキーフレーム間を補間する。

- 06:00：朝
- 09:00：午前
- 12:00：昼
- 15:00：午後
- 18:00：夕方終了
- 21:00：夏祭りの夜

補間対象は、空上部・空下部・地平線・地面・建物・影・太陽または月・星・街灯・窓・暖色オーバーレイ。実際のゲーム時計が実装された後も同じ関数を使うため、M0の演出は使い捨てにしない。

## マップストリーミング方針

M1から、屋外マップを複数チャンクへ分割する。

- 現在のチャンク
- 隣接チャンク
- 進行方向の先読みチャンク

だけを保持し、遠いチャンクを破棄する。プレイヤーからは一つの町として見え、通常のエリア境界でロード画面を表示しない。

## デプロイ方針

1. 一時ブランチで実装
2. `npm run check`
3. PR作成
4. GitHub Actions成功
5. `main`へsquash merge
6. VercelがProduction Deploymentを作成
7. 本番URLで確認

恒久的な`develop`ブランチ、ステージング環境、開発専用データベースは作らない。
