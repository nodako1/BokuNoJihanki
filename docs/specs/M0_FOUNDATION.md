# M0 開発基盤 仕様書

作成日：2026-07-22  
対象：`nodako1/BokuNoJihanki`  
マイルストーン：M0  
バージョン：0.0.1

## 1. 目的

今後の全機能を安全に積み重ねられるWebゲーム基盤を構築し、スマホ横画面でVercelへアクセスした時点で「この町で夏休みを過ごすゲーム」であることが伝わる状態にする。

M0では本編の移動・自販機探索を作らない。代わりに、以下を使い捨てではない共通基盤として完成させる。

- ReactとPhaserの共存
- スマホ横画面
- PWAの基本設定
- タイトル画面
- ユーザー操作後の音声開始
- 時間帯による町の連続変化
- ビルド情報
- CIとVercel設定
- 仕様・進捗のリポジトリ管理

## 2. M0の画面

### 2.1 初期表示

1. 夏の町の朝を背景表示する。
2. 左側にタイトルカードを表示する。
3. タイトルは「ぼくの自販機」。
4. キャッチコピーは「人生で一番バカで、一番楽しい夏休み。」。
5. 「夏休みを始める」ボタンを表示する。
6. 音は再生しない。

### 2.2 開始後

1. オリジナルBGM、夏の環境音、開始効果音を再生する。
2. 朝6:00から夏祭りの夜21:00まで、15分ずつ自動で進める。
3. 町の空、太陽・月、星、建物、地面、窓、自販機、街灯が連続変化する。
4. 左上に8月1日、時刻、時間帯を表示する。
5. 右上にM0プレビュー表示を置く。
6. 下部に次の確認操作を置く。
   - 15分進める
   - 時間を止める／流す
   - 朝へ戻す
   - 音をON／OFF

### 2.3 縦向き

画面全体を横向き案内で覆い、ゲーム操作を受け付けない。端末を横向きにすると自動で解除する。

## 3. 時間帯仕様

### 3.1 基本時刻

- 一日の開始：6:00
- 通常日の終了：18:00
- 特別日の終了：21:00
- 行動単位：15分

M0の自動再生は技術確認用であり、本編ではプレイヤー行動によって時刻を進める。

### 3.2 フェーズ

| 時刻 | フェーズ | 主な変化 |
|---|---|---|
| 6:00〜8:59 | 朝 | 黄色い朝日、淡い空、柔らかい光 |
| 9:00〜14:59 | 昼 | 鮮明な青空、強い日差し、白い雲 |
| 15:00〜17:59 | 夕方 | 橙色の地平線、暖色、影の深まり |
| 18:00〜21:00 | 夏祭りの夜 | 濃紺、月、星、街灯、窓、自販機の光 |

フェーズ境界で画像を突然交換せず、複数のキーフレームを色・位置・透明度で補間する。

### 3.3 本編への継承

`getAtmosphere(minutes)`を本編のゲーム時計から呼び出す。M0のプレビューコードをM2以降も再利用し、時間帯システムを作り直さない。

## 4. 音声仕様

### 4.1 ブラウザー制約への対応

ユーザーが「夏休みを始める」を押すまでAudioContextを作成しない。

### 4.2 M0音源

第三者素材を使わず、Web Audio APIで実行時生成する。

- 低音の柔らかいパッド
- 短いペンタトニック旋律
- セミを想起させる帯域ノイズ
- 開始効果音
- ボタンクリック効果音

時間帯に応じてノイズ帯域、音量、旋律の高さを変える。M8で本番BGMへ置き換える場合も、音声開始・ミュート・時間帯連動の基盤は維持する。

## 5. 技術仕様

### 5.1 ランタイム

- Node.js 22
- React 19.0.0
- Phaser 4.2.1
- TypeScript 5.7.2
- Vite 6.3.1

### 5.2 基準解像度

- 論理解像度：1280×720
- アスペクト比：16:9
- 拡縮：FIT
- 中央配置：CENTER_BOTH
- アンチエイリアス：有効
- ドット絵モード：無効

### 5.3 PWA

- `display: standalone`
- `orientation: landscape`
- M0ではService Workerを導入しない
- Vercel更新後に古いファイルが残るリスクを避ける
- 正式版前にオフライン対応と更新通知を追加する

## 6. CI/CD仕様

### 6.1 GitHub Actions

PRおよび`main`へのPushで以下を実行する。

1. Node.js 22セットアップ
2. `npm install`
3. プロジェクト構成検証
4. ESLint
5. TypeScript型チェック
6. Node標準テスト
7. Vite本番ビルド
8. `dist`を成果物として保存

### 6.2 Vercel

- Framework：Vite
- Build：`npm run build`
- Output：`dist`
- Production Branch：`main`
- SPAリライトを設定
- ハッシュ付き`assets`だけ長期キャッシュ
- HTMLとmanifestは再検証可能にする

### 6.3 ビルド識別

画面右下に次を表示する。

- アプリバージョン
- コミットSHA先頭7文字
- マウス・長押し時にビルド日時
