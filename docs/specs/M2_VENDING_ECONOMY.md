# M2仕様 自販機探索と経済（ドラフト）

作成: 2026-07-23 クロード（Claude）。ロードマップM2と README「未実装」を基に作成したドラフトです。数値は仮案で、Koichiさんの承認とチャッピーの意見で確定します。

## 完了条件（ロードマップより）

「自販機を見つけ、調べ、結果を確認し、15分と所持金が動き、再読み込み後も復元できる」一連の中心体験がProductionの実ブラウザーで動作すること。

## スコープ

- 自販機への接近検知と状況対応ボタン（「したをのぞく」「おつりぐちをみる」）
- 接写画面（自販機の下／お釣り返却口）と結果演出
- 固定乱数による金額抽選（空振り〜1,000円）
- 所持金（Wallet）と加算処理
- 探索1回につき15分の時刻経過
- 同じ自販機・同じ調べ方は1日1回まで
- 18:00以降は探索不可（時刻チェック）
- ローカルセーブ基盤（所持金・時刻・当日の探索履歴を再読み込み後に復元）

## 非スコープ（M3以降）

日付変更、自室・帰宅、日記、ゲームショップ、購入、NPC、複数日進行。

## アーキテクチャ方針

M1.3が住宅街シーンを再構築中のため、M2コアはPhaser非依存のピュアTypeScriptとして`src/game/economy/`に新設し、シーン統合はM1.3マージ後に行う二段階構成とする。

### フェーズA: コアロジック（シーン非依存・先行実装）

- `src/game/economy/wallet.ts` — 所持金の保持・加算・減算・上限
- `src/game/economy/gameClock.ts` — ゲーム内時刻の取得・15分消費・18:00判定（既存の時間帯システムの読み取りインターフェースに接続するアダプター前提）
- `src/game/economy/searchSystem.ts` — 探索実行の判定（当日1回・18時制限）と結果確定
- `src/game/economy/lootTable.ts` — 抽選テーブルと固定乱数
- `src/game/economy/rng.ts` — シード式PRNG（mulberry32等の決定論的実装）
- `src/game/economy/saveData.ts` — localStorageスキーマ、バージョン、シリアライズ／復元
- `tests/` — 上記すべてのユニットテスト（抽選分布、当日1回、18時制限、セーブ往復）

### フェーズB: シーン統合（M1.3マージ後）

自販機オブジェクト配置、接近判定、アクションボタンUI、接写画面、結果演出、HUDの所持金表示。M1.3で確定するワールドAPIに合わせて設計し、担当は連携ボードで相談して決める。

## 固定乱数の仕様（仮）

- シードは「ゲームセーブ固有シード＋日付＋自販機ID＋調べ方」から生成し、同じ条件では同じ結果を返す（リロードによる引き直し不可）
- 抽選テーブル仮案（要承認）:

| 結果 | 下をのぞく | お釣り返却口 |
| --- | --- | --- |
| 空振り | 55% | 60% |
| 10円 | 20% | 25% |
| 50円 | 10% | 8% |
| 100円 | 10% | 5% |
| 500円 | 4% | 1.5% |
| 1,000円札 | 1% | 0.5% |

- 期待値はプレイ全体の経済（M4のエリア期待値・M7の在庫93本）と合わせて後日調整する

## セーブデータ（仮スキーマ）

```json
{
  "version": 1,
  "seed": 123456789,
  "wallet": 0,
  "clock": { "day": 1, "minutes": 360 },
  "searchedToday": [{ "machineId": "res-west-01", "action": "under" }]
}
```

キー名: `boku-no-jihanki:save:v1`。将来のバージョン移行関数を最初から用意する。

## 受け入れテスト（フェーズA）

- 同一シード・同一条件で抽選結果が一致する
- 分布テスト: 大量試行で仮テーブルの確率に収束する
- 同じ自販機・同じ調べ方の2回目は当日拒否される
- 17:46〜18:00をまたぐ探索の扱い（18:00以降開始不可、開始済みは完了扱い）※要相談
- セーブ→復元で所持金・時刻・探索履歴が完全一致
- `npm run check`（validate、lint、typecheck、test、build）成功
