**ランタイム修正完了・Vercel Production再確認待ち**

- バージョン: `0.1.0`
- 初回M1機能マージ: `432108024b40c9f3fc20aeec38b7bf871c192da1`

- 障害: タイトル開始後、UIは表示されるがPhaser Canvasが黒く、HUDが0 FPS・座標0,0・チャンク0件になる
- 原因: percent-encoded SVG Data URIへPhaserローダーが`atob`を適用し、Scene起動前に`InvalidCharacterError`となる
- 修正: SVGをUTF-8対応base64 Data URIへ変換して読み込む

## マップ

ワールドは5120×720。道路の通行帯は主にY=559〜694、公園の園路はY=552〜694。プレイヤーの可動範囲はY=356〜690。右へ進むほど住宅街から公園へ移る。

内部チャンクは以下の4つです。

1. `residential-west`
2. `residential-east`
3. `park-west`
4. `park-east`

現在地、隣接、進行方向のチャンクを読み込み、十分に離れたチャンクを破棄します。

## 品質・本番確認

`npm run check`による構成検証、ESLint、TypeScript、Node標準テスト、Vite本番ビルドだけでは完了としません。Pull Requestでは`Browser Smoke`、mainマージ後はVercel Productionへの`Production Smoke`を実行し、実ブラウザーで次を確認します。

1. タイトルとCanvasが起動する
2. 「夏休みを始める」後にFPSが0より大きくなる
3. 初期座標と住宅街チャンクが読み込まれる
4. JavaScript例外と主要リソース失敗がない
5. 矢印キーで主人公が移動する
6. 住宅街から`park-west`へロード無しで到達する
7. Canvasが黒一色ではない
8. HUDのBuild SHAがVercel対象コミットと一致する

旧`Production Smoke`は公開バンドルの文字列だけを検査しており、Scene起動前の例外を見逃しました。今後は実操作結果のみをProduction完了根拠とします。
